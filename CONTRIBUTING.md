# Contributing

Thanks for your interest in improving this documentation.

## Ways to Contribute

1. **Fix errors**: Typos, outdated commands, broken links
2. **Clarify explanations**: If something confused you, it probably confuses others
3. **Add troubleshooting tips**: Share issues you encountered and how you solved them
4. **Improve formatting**: Better structure, clearer headings, more useful diagrams

## How to Contribute

1. Fork the repository
2. Create a branch for your changes
3. Make your edits
4. Submit a pull request with a clear description

## Guidelines

- Keep the personal narrative style (first person: "I did X")
- Focus on practical, actionable content
- Include verification steps where possible
- Test any commands before submitting

## Questions?

Open an issue if you're unsure about something or want to discuss a larger change before starting.
